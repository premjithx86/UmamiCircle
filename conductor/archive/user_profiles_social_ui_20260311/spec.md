# Specification: User Profiles and Social UI

## Overview
This track implements the user profile pages and the UI for social connections. It focuses on the visual representation of user data, following relationships, and the social activity feed.

## Objectives
1. Build the User Profile page for both personal and other users' views.
2. Implement social relationship UI (Follow/Unfollow buttons, lists of followers/following).
3. Create the user's personal activity feed.

## Functional Requirements
- **Profile Page:**
    - Profile header (cover photo, avatar, bio, stats).
    - Grid/List view of user's posts and recipes.
    - Profile editing functionality (frontend).
- **Social Lists:**
    - Modal/Page for viewing "Followers" and "Following".
    - Searchable list of connections.
- **Relationship Controls:**
    - Follow/Unfollow buttons with real-time feedback.
    - Block/Unblock buttons.
- **Activity Feed:**
    - View recent social actions (followed X, liked Y).

## Success Criteria
- Profiles are rendered correctly based on URL username.
- Users can toggle follow status with visual updates.
- Followers and Following lists are accessible and accurate.
- Profile editing (username, bio, avatar) works as expected.
